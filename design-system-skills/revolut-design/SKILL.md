# Revolut Design System

A bold fintech system pairing a stark black canvas with the brand's electric indigo-blue, high-contrast type, and a broad palette of functional accents for data and status.

## When to use
Apply when building or restyling any fintech, banking, payments, or finance product interface that should feel like Revolut.

## Core tokens (cheat-sheet)
- **Primary (indigo)**: `#494fdf` (bright `#4f55f1`, deep `#3a40c4`) — primary CTAs and brand voltage
- **Canvas**: dark `#000000` / light `#ffffff`; surfaces `#0a0a0a`, elevated `#16181a`, soft `#f4f4f4`
- **Ink**: `#191c1f` / body `#1f2226` / mute `#505a63` / stone `#8d969e`; on-dark `#ffffff` (mute `rgba(255,255,255,0.72)`)
- **Hairlines**: light `#e2e2e7`, dark `rgba(255,255,255,0.12)`
- **Functional accents**: teal `#00a87e`, warning `#ec7e00`, pink `#e61e49`, danger `#e23b4a`, yellow `#b09000`, link `#376cd5` — for data, status, and charts
- **Type**: high-contrast grotesk; bold confident headlines (see reference for exact stack and scale)

## How to apply
1. Read `reference/DESIGN.md` for the full token set, typography scale, component specs, light/dark surface rules, and responsive behavior.
2. Use the indigo as the primary brand voltage; reserve the functional accent palette for data, status, and category coding.
3. Respect the dual light/dark canvas system.

The complete specification lives in `reference/DESIGN.md`.
