# Airtable Design System

A sober, editorial workflow-software interface on white canvas with dark-ink type, where brand voltage comes from full-bleed signature cards in coral, dark green, peach, and navy. Type runs in modest weights — never bold for its own sake.

## When to use
Apply when building or restyling any workflow/productivity B2B SaaS interface or long-scroll explainer page that should feel like Airtable.

## Core tokens (cheat-sheet)
- **Primary**: near-black `#181d26` (active `#0d1218`) — pill CTA. Secondary actions: white outlined button.
- **Canvas**: white `#ffffff`; soft `#f8fafc`; strong `#e0e2e6`; dark `#181d26` / elevated `#1d1f25`
- **Ink**: `#181d26` / body `#333840` / muted `#41454d`
- **Hairlines**: `#dddddd`; strong border `#9297a0`
- **Signature cards (full-bleed)**: coral `#aa2d00`, forest `#0a2e0e`, cream `#f5e9d4`, peach `#fcab79`, mint `#a8d8c4`, yellow `#f4d35e`, mustard `#d9a441`
- **Semantic**: link `#1b61c9`, info `#254fad`, success `#006400`
- **Type**: Haas Grotesk (sub with a neutral grotesk) in modest weights

## How to apply
1. Read `reference/DESIGN.md` for the full token set, typography scale, component specs, and responsive rules.
2. Keep chrome sober and editorial; let the saturated signature cards punctuate long-scroll pages.
3. Primary CTA is a near-black pill; secondary is a white outlined button.

The complete specification lives in `reference/DESIGN.md`.
